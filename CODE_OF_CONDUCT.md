# Code Of Conduct

## Our Standard

Warlog Spec is a technical standards project. We want discussions to be
direct, reviewable, and respectful so implementers can disagree about the
contract without making the project hostile.

Expected behavior includes:

- giving specific technical feedback
- explaining assumptions and tradeoffs
- welcoming contributors with different levels of experience
- challenging claims with evidence rather than personal attacks
- respecting maintainer decisions after discussion has run its course

Unacceptable behavior includes:

- harassment, intimidation, or discriminatory language
- personal attacks, insults, or sustained bad-faith argument
- publishing private information without permission
- leaking secrets, credentials, exploit details, or private tenant data
- disrupting issues, RFCs, reviews, or release work

## Scope

This code applies in project spaces, including issues, pull requests,
RFC discussions, community channels, events, and public conversations
where someone is representing the project.

## Enforcement

Maintainers may remove, edit, hide, or reject comments, issues, pull
requests, commits, and other contributions that violate this code. They
may also temporarily or permanently restrict participation when needed to
protect the project and its contributors.

## Reporting

Report unacceptable behavior privately to the maintainers through the
channels listed in `GOVERNANCE.md`. Security-sensitive reports should use
the process in `SECURITY.md` instead.

Reports will be handled with appropriate confidentiality. Maintainers are
expected to avoid conflicts of interest when reviewing conduct reports.
